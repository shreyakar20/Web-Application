Definition

The Controller acts as an intermediary between the Model and the View. It receives user requests, processes them, updates the Model, and returns the appropriate View.

Responsibilities

  • Receives user requests
  • Processes input
  • Calls the Model
  • Updates the View
  • Controls application flow

MVC Workflow

  1. User sends request through the View.
  2. Controller receives the request.
  3. Controller interacts with the Model.
  4. Model processes the data.
  5. Controller receives updated data.
  6. View displays the updated result.
